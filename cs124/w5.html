Weekly Progress

This week marked the beginning of the actual coding for the project. I began coding the basic sequencer that will be eventually be able to handle insertions as well.

One set of functions that I wrote is responsible for representing DNA reads in the most concise way possible. Each read is assigned a number based on its nucleotide sequence. Cytosine corresponds to 0, guanine to 1, adenine to 2, and thymine to 4. We then pretend that a DNA read is a base-4 number, and store its base-10 equivolent. 



#define CYTOSINE 0
#define GUANINE 1
#define ADENINE 2
#define THYMINE 3


/**
 * Given a DNA read in string format (e.g. "AACTGCA..."),
 * encode it as a base-4 number with:
 *     C = 0
 *     G = 1
 *     A = 2
 *     T = 3
 * and return the encoded value.
 *
 * For the scope of this project, read lengths will not
 * exceed 30 base pairs. This corresponds to 4^30 (or about
 * 10^18) such sequences, so we will use a long long
 * integer to return them.
 */
long long SequenceRead::encodeRead(string read) {
	long long r = 0;

	for (int i = 0; i < i < m_length; i++) {
		r *= 4;
		switch(read[i]) {
			case 'C': r += CYTOSINE; break;
			case 'G': r += GUANINE; break;
			case 'A': r += ADENINE; break;
			case 'T': r += THYMINE; break;
			default: return -1;
		}
	}

	return r;
}

Using this method (and a similar method for decoding), the SequenceRead class contains all of the necessary information pertaining to a single sequence. These are used to create the ReferenceGenome class, which is nothing more than a hash table on all possible sequences (currently I am using length 5, and will eventually extend perhaps as high as 30, although this may push the memory limits of my computer).
