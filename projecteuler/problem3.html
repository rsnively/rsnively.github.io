Problem 3: Largest Prime Factor

 

 

The Problem

The prime factrs of 13195 are 5, 7, 13, and 29.

What is the largest prime factor of the number 600851475143?

 

The Solution

Spoiler Alert: This website looooooves to make puzzles about prime numbers. Eventually, the become very zealous in their efforts to make us get creative. For this problem, however, we can throw together the sloppiest of prime number tests and still get the answer somewhat quickly (in 32ms on my machine).

 

Never admit to writing code like this: 


#include <iostream>
using namespace std;

const long long number = 600851475143;

// This constant is important in limiting the number of numbers we search
// through. We know that the largest possible factor of any number is its
// square root, so it's pointless to look at numbers beyond this one.
const long long square = pow(number, 0.5);
                           
int main()
{
    // What will eventually contain our largest prime factor.
    long long max = 0;
    
    // Optimizing all the way, we start from 3 because except for 2, there is
    // no prime factor which is even (thus we also increment by 2 instead of 1).
    for (long long i = 3; i <= square; i += 2)
    {
        // First we see if our number is divisible by this index.
        if (number % i == 0) 
        {
            // Now we use the grossest, most disgusting means of determining if
            // i is prime. Rated PG-13.
            bool prime = true;
            long long square_i = pow(i, 0.5);
            for (long long j = 2; prime && j < square_i; j++)
            {
                if (i % j == 0)
                {
                    prime = false;
                    break;
                }
            }
            
            if (prime)
            {
                max = i;
            }
        }
    }
    
    cout << max << endl;
}

 

There are many more efficient ways to determine if a number is prime. My method of choice is utilising the Sieve of Erasmus (coming soon to a Project Euler solution near you). Since this problem deals with multiples however, we don't really even need to check if a number is prime. Rather, we just divide our really big number by everything we can until we have its largest prime factor.

I'm just going to say it: I think this is a dumb solution. If the problem had to do with only finding one Fibonacci number, then sure, a closed-form solution would blow a standard one out of the water. However, the loop is only reduced by a factor of three (and the I'm pretty sure one of the 10 Commandments says something about scalar reductions in complexity being made-up). Not only that, but this method requires not one, but two function calls for each number. The stack traces probable take a lot longer than running through two extra iterations of the loop. This problem is certainly an instance of everyone getting too obsessed with proving their math acumen.
